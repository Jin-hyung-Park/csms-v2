## 🔍 추가 확인 권장 사항

### 1. 보호된 페이지 접근 확인

- [ ] 로그인 없이 `/employee/dashboard` 접근 시 `/login`으로 리다이렉트 되는지 확인
- [ ] 로그인 후 보호된 페이지 접근이 정상 동작하는지 확인
- [ ] 로그아웃 후 보호된 페이지 접근 시 `/login`으로 리다이렉트 되는지 확인

### 2. 토큰 관리 확인

- [ ] 로그인 후 localStorage에 토큰이 저장되는지 확인
  - 브라우저 개발자 도구 → Application → Local Storage → `token` 확인
- [ ] API 요청 시 자동으로 토큰이 헤더에 추가되는지 확인
  - 브라우저 개발자 도구 → Network → 요청 헤더 → `Authorization: Bearer <token>` 확인

### 3. WorkSchedule API 동작 확인

- [ ] 로그인 후 근무일정 조회 (`/employee/schedule`) 동작 확인
- [ ] 근무일정 생성 기능 동작 확인
- [ ] 근무일정 수정 기능 동작 확인
- [ ] 근무일정 삭제 기능 동작 확인
- [ ] 토큰 없이 WorkSchedule API 접근 시 401 에러 반환 확인

### 4. 에러 처리 확인

- [ ] 잘못된 이메일/비밀번호 로그인 시 에러 메시지 표시 확인
- [ ] 중복 이메일 회원가입 시 에러 메시지 표시 확인
- [ ] 토큰 만료/유효하지 않은 토큰 시 자동 로그아웃 및 리다이렉트 확인

### 5. 권한 확인

- [ ] 근로자로 로그인 시 자신의 근무일정만 조회되는지 확인
- [ ] 근로자가 다른 사용자의 근무일정을 수정/삭제하려고 할 때 권한 에러 발생 확인
