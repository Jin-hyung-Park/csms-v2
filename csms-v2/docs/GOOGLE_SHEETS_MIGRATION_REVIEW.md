# Google Sheets → csms-v2 근무 데이터 마이그레이션 검토

> **대상 스프레드시트**: [1월 급여 계산](https://docs.google.com/spreadsheets/d/1xsaJjfWFLMXZyN8fMfl71lcnCCPvi63oW4j92G8YeYk/edit?usp=sharing)  
> **대상 시트**: `대치메가 1월 근무`, `삼성메가 1월 근무` (1월 근무 데이터)  
> **목표**: csms-v2의 **WorkSchedule**(근무 일정) 컬렉션으로 이관

---

## 1. 결론: **가능함 (전제 조건 충족 시)**

- Google Sheets는 “링크가 있는 모든 사용자”로 공개되어 있으므로 **API 키만으로** 시트 값을 읽을 수 있습니다.
- csms-v2에는 이미 **WorkSchedule** 모델과 API가 있으므로, 시트 데이터를 파싱해 **일괄 삽입**하는 스크립트를 추가하면 마이그레이션이 가능합니다.
- 다만 **점포(Store)·직원(User) 매핑**과 **시트 컬럼 구조**를 먼저 정해야 합니다.

---

## 2. csms-v2 근무 데이터 구조 (WorkSchedule)

| 필드 | 타입 | 필수 | 설명 |
|------|------|------|------|
| `userId` | ObjectId (User) | ✅ | 근무한 직원 |
| `storeId` | ObjectId (Store) | ✅ | 소속 점포 |
| `workDate` | Date | ✅ | 근무일 |
| `startTime` | String (HH:MM) | ✅ | 출근 시각 |
| `endTime` | String (HH:MM) | ✅ | 퇴근 시각 |
| `totalHours` | Number | (자동) | start/end 기준 자동 계산 |
| `status` | enum | ✅ | `pending` \| `approved` \| `rejected` (기본: pending) |
| `notes` | String | | 메모 (선택) |

- **인덱스**: `userId + workDate`, `storeId + workDate` 등으로 조회 최적화됨.
- **API**: `POST /api/work-schedules` 로 1건 생성 가능. 대량 이관은 **스크립트에서 `WorkSchedule.insertMany()`** 사용 권장.

---

## 3. 마이그레이션에 필요한 매핑

### 3.1 점포(Store) 매핑

| 시트 이름 | csms-v2 Store |
|-----------|----------------|
| 대치메가 1월 근무 | “대치메가” 또는 동일 이름의 Store 1개 |
| 삼성메가 1월 근무 | “삼성메가” 또는 동일 이름의 Store 1개 |

- 각 시트의 데이터는 **해당 Store의 `_id`(storeId)** 와 연결해야 합니다.
- csms-v2에 이미 “대치메가”, “삼성메가” 점포가 있으면 해당 `_id` 사용.
- 없으면 마이그레이션 전에 **Store를 먼저 생성**하거나, 스크립트에서 “점포명 → storeId” 매핑 테이블을 두고 처리.

### 3.2 직원(User) 매핑

- 시트에는 **이름**(및 주민번호 등)만 있고, csms-v2에는 **User 문서(`_id`, email, name, storeId)** 가 있습니다.
- 따라서 **“이름 + 점포(또는 시트)” → `userId`** 로 매핑이 필요합니다.
  - 동명이인이 있을 수 있으면 **주민번호/입사일** 등으로 보조 매핑하는 것이 안전합니다.
- 옵션:
  1. **이미 등록된 직원만 이관**: 시트의 “이름”과 csms-v2 `User.name` + `User.storeId` 로 매칭해 `userId` 사용.
  2. **미등록자는 제외 또는 목록 출력**: 매칭 실패 건은 로그/CSV로 남기고, 나중에 수동 등록 후 재실행 또는 수동 입력.

## 4. 시트 데이터 형식 가정 (확인 필요)

“대치메가 1월 근무”, “삼성메가 1월 근무” 시트의 **실제 컬럼**은 확인이 필요합니다. 아래는 일반적인 “근무 시트” 형태 예시입니다.

### 4.1 필요한 정보 (최소)

- **날짜**: 근무일 (예: 2026-01-05 또는 1/5)
- **직원 식별**: 이름 (필수), 가능하면 주민번호 등
- **시간**: 시작 시각, 종료 시각 (예: 15:00, 18:30 또는 “15:00~18:30” 한 셀)

### 4.2 예시 컬럼 (시트에 맞게 수정 필요)

| 날짜 | 이름 | 시작 | 종료 | 비고 |
|------|------|------|------|------|
| 2026-01-01 | 김현지 | 15:00 | 18:30 | |
| 2026-01-02 | 변주안 | 22:30 | 06:30 | 심야 |

- “근무시간”이 **한 셀에 "15:00~18:30"** 형태라면, 스크립트에서 `~` 또는 `-`로 split 해 `startTime`/`endTime`으로 나누면 됩니다.
- **날짜**가 “1/5”, “1월 5일” 등이면 연도(2026)와 합쳐서 `workDate` (Date)로 파싱해야 합니다.

## 5. 구현 방안

### 5.1 Google Sheets에서 데이터 읽기

- **스프레드시트 ID**: `1xsaJjfWFLMXZyN8fMfl71lcnCCPvi63oW4j92G8YeYk`
- **시트 이름(범위)**:
  - `대치메가 1월 근무` → 범위 예: `'대치메가 1월 근무'!A1:Z500`
  - `삼성메가 1월 근무` → 범위 예: `'삼성메가 1월 근무'!A1:Z500`

**방법 A – Google Sheets API v4 (권장)**

- 공개 스프레드시트이므로 **API 키만**으로 읽기 가능.
- Node 예시:
  - `GET https://sheets.googleapis.com/v4/spreadsheets/{spreadsheetId}/values/{range}?key={API_KEY}`
  - 또는 `googleapis` 패키지의 `sheets.spreadsheets.values.get()` 사용.
- GCP 콘솔에서 “Google Sheets API” 사용 설정 후 API 키 생성 필요.

**방법 B – CSV 내보내기**

- 시트를 CSV로 내보낸 뒤, Node 스크립트에서 `fs` + `csv-parse` 등으로 읽어서 파싱. API 설정 없이 가능하지만, 시트별로 수동 내보내기 필요.

### 5.2 마이그레이션 스크립트 구성 제안

1. **설정**
   - 스프레드시트 ID, 시트 이름 2개, (선택) API 키 또는 CSV 경로.
   - “대치메가” → `storeId`, “삼성메가” → `storeId` 매핑 (또는 이름으로 DB 조회).
2. **데이터 수집**
   - 위 두 시트에서 행 단위로 읽어, 각 행을 `{ date, name, startTime, endTime, notes? }` 형태로 정규화.
3. **매핑**
   - `name` + 시트(점포)로 User 조회 → `userId`. 없으면 스킵 또는 목록 저장.
   - 시트별로 `storeId` 부여.
4. **저장**
   - `WorkSchedule` 문서 배열을 만든 뒤 `WorkSchedule.insertMany()` (중복 방지를 위해 `userId + workDate + startTime` 등으로 기존 데이터 검사 후 삽입 가능).
5. **결과**
   - 삽입 건수, 스킵(매칭 실패) 건수, 에러 로그 출력.

## 6. 전제 조건 및 주의사항

| 항목 | 내용 |
|------|------|
| **점포 존재** | “대치메가”, “삼성메가”에 해당하는 Store가 csms-v2에 있어야 함. 없으면 먼저 생성. |
| **직원 등록** | 이관할 직원이 User로 등록되어 있고, `storeId`가 해당 점포로 설정되어 있어야 함. |
| **시트 컬럼** | “대치메가 1월 근무”, “삼성메가 1월 근무”의 실제 컬럼명/순서를 확인한 뒤, 파서를 그에 맞게 작성해야 함. |
| **시간 형식** | HH:MM 또는 “HH:MM~HH:MM” 등 일관된 형식이어야 파싱 가능. 24시간제 권장. |
| **중복** | 같은 userId, workDate, startTime 조합이 이미 있으면 스킵 또는 업데이트 정책 결정 필요. |
| **status** | 과거 근무는 마이그레이션 시 `status: 'approved'` 로 두는 것이 무난함. |

   - `server/scripts/migrate-work-from-google-sheets.js` 같은 마이그레이션 스크립트를 추가하고,  
   - 위 조건이 정리되면 실제 시트 구조에 맞춰 파싱 로직과 `insertMany` 로직을 구현하면 됩니다.

정리하면, **Google Sheets의 “대치메가 1월 근무”, “삼성메가 1월 근무” 데이터를 csms-v2 근무(WorkSchedule) 데이터로 마이그레이션하는 것은 가능**하며, 시트 구조와 Store/User 매핑만 확정되면 구체적인 스크립트까지 작성할 수 있습니다.
