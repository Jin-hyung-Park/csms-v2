# 카카오 로그인 설정 가이드

## 1. 카카오 개발자 계정 설정

### 1.1 카카오 개발자 계정 생성
1. [Kakao Developers](https://developers.kakao.com/)에 접속
2. 카카오 계정으로 로그인
3. "내 애플리케이션" → "애플리케이션 추가하기" 클릭

### 1.2 애플리케이션 정보 설정
- **앱 이름**: CSMS
- **사업자명**: 개인 또는 회사명
- **카테고리**: 비즈니스/경제

### 1.3 플랫폼 설정
1. **Web 플랫폼 추가**
   - 사이트 도메인: `http://localhost:3000`
   - JavaScript 키 복사 (나중에 사용)

2. **카카오 로그인 활성화**
   - "카카오 로그인" → "활성화 설정" → "활성화"
   - Redirect URI: `http://localhost:3000/auth/kakao/callback`

## 2. 환경 변수 설정

### 2.1 서버 환경 변수 (.env)
```bash
# 기존 설정에 추가
KAKAO_CLIENT_ID=your-kakao-client-id
KAKAO_CLIENT_SECRET=your-kakao-client-secret
KAKAO_REDIRECT_URI=http://localhost:3000/auth/kakao/callback
```

### 2.2 클라이언트 환경 변수 (client/.env)
```bash
REACT_APP_API_URL=http://localhost:5001/api
REACT_APP_KAKAO_APP_KEY=your-kakao-javascript-key
```

## 3. 카카오 로그인 기능

### 3.1 주요 기능
- **카카오 계정으로 가입**: 새로운 사용자는 카카오 계정으로 자동 가입
- **기존 계정 연동**: 이미 가입된 이메일로 카카오 계정 연동 가능
- **자동 로그인**: 카카오 토큰을 통한 자동 로그인

### 3.2 사용자 플로우
1. 로그인/회원가입 페이지에서 "카카오톡으로 시작하기" 버튼 클릭
2. 카카오 로그인 팝업에서 카카오 계정으로 로그인
3. 사용자 정보 동의 (닉네임, 이메일, 프로필 이미지)
4. 자동으로 시스템에 로그인/가입 완료

### 3.3 데이터 처리
- **카카오 ID**: 고유 식별자로 사용
- **이메일**: 기존 계정과 연동 시 사용
- **닉네임**: 사용자명으로 설정
- **프로필 이미지**: 프로필 사진으로 저장

## 4. 보안 고려사항

### 4.1 토큰 관리
- 카카오 액세스 토큰은 서버에서 검증 후 JWT 토큰 발급
- 클라이언트에서는 JWT 토큰만 저장

### 4.2 데이터 검증
- 서버에서 카카오 사용자 정보 검증
- 중복 가입 방지 (이메일, 카카오 ID 기준)

## 5. 테스트 방법

### 5.1 개발 환경 테스트
1. 서버와 클라이언트 모두 실행
2. 카카오 개발자 콘솔에서 앱 상태 확인
3. 로그인 페이지에서 카카오 로그인 버튼 테스트

### 5.2 프로덕션 배포 시
1. 카카오 개발자 콘솔에서 도메인 추가
2. 환경 변수 업데이트
3. HTTPS 설정 확인

## 6. 문제 해결

### 6.1 일반적인 오류
- **"카카오 SDK가 로드되지 않았습니다"**: JavaScript 키 확인
- **"카카오 로그인에 실패했습니다"**: Redirect URI 설정 확인
- **"이미 등록된 이메일입니다"**: 기존 계정과 연동 시도

### 6.2 디버깅
- 브라우저 개발자 도구에서 콘솔 로그 확인
- 네트워크 탭에서 API 요청/응답 확인
- 카카오 개발자 콘솔에서 로그 확인
