# 모바일 환경 테스트 가이드

## 개요
이 가이드는 편의점 근무 관리 시스템(CSMS)을 모바일 환경에서 테스트하는 방법을 설명합니다.

## 사전 준비

### 1. 개발 환경 설정
```bash
# 프로젝트 클론 (아직 하지 않았다면)
git clone <repository-url>
cd convenience_store_management

# 의존성 설치
npm run install:all

# 환경 변수 설정
cp env.example .env
cp client/env.example client/.env
```

### 2. 환경 변수 설정

#### 서버 환경 변수 (.env)
```bash
NODE_ENV=development
PORT=5001
MONGODB_URI=mongodb://localhost:27017/convenience_store
JWT_SECRET=your-super-secret-jwt-key-here
JWT_EXPIRE=7d
KAKAO_CLIENT_ID=your-kakao-client-id
KAKAO_CLIENT_SECRET=your-kakao-client-secret
KAKAO_REDIRECT_URI=http://172.16.0.91:3000/auth/kakao/callback
```

#### 클라이언트 환경 변수 (client/.env)
```bash
REACT_APP_API_URL=http://172.16.0.91:5001/api
REACT_APP_KAKAO_APP_KEY=your-kakao-javascript-key
```

## 모바일 테스트 실행

### 1. 모바일 접근용 서버 실행
```bash
# 모바일 테스트용 개발 서버 실행
npm run dev:mobile
```

### 2. 네트워크 정보 확인
```bash
# Mac/Linux
ifconfig | grep "inet " | grep -v 127.0.0.1

# Windows
ipconfig
```

### 3. 모바일에서 접근
- 모바일 브라우저에서 `http://172.16.0.91:3000` 접속
- 같은 Wi-Fi 네트워크에 연결되어 있어야 함

## 테스트 시나리오

### 시나리오 1: 근로자 로그인 테스트
1. **카카오 로그인 테스트**
   - 모바일에서 카카오톡 앱이 설치되어 있는지 확인
   - "카카오톡으로 시작하기" 버튼 클릭
   - 카카오 로그인 팝업에서 계정 선택
   - 로그인 성공 후 대시보드 이동 확인

2. **이메일 로그인 테스트**
   - 이메일과 비밀번호 입력
   - 로그인 버튼 클릭
   - 로그인 성공 후 대시보드 이동 확인

### 시나리오 2: 근무시간 입력 테스트
1. **근무시간 입력**
   - 대시보드에서 "근무시간 입력" 메뉴 클릭
   - 날짜 선택 (모바일 터치 인터페이스 테스트)
   - 시작 시간 선택 (5분 단위 선택 확인)
   - 종료 시간 선택
   - 저장 버튼 클릭

2. **입력 데이터 검증**
   - 잘못된 시간 입력 시 오류 메시지 표시 확인
   - 중복 시간 입력 시 경고 메시지 확인
   - 저장 성공 시 확인 메시지 표시

### 시나리오 3: 모바일 UI/UX 테스트
1. **반응형 디자인 확인**
   - 세로/가로 모드 전환 시 레이아웃 적응 확인
   - 다양한 화면 크기에서 UI 요소 배치 확인
   - 터치 인터페이스 최적화 확인

2. **터치 인터랙션 테스트**
   - 버튼 터치 반응성 확인
   - 스크롤 동작 확인
   - 입력 필드 포커스/블러 동작 확인

### 시나리오 4: 알림 기능 테스트
1. **알림 확인**
   - 알림 아이콘 클릭
   - 알림 목록 스크롤 확인
   - 알림 읽음 처리 확인

## 문제 해결

### 일반적인 문제들

#### 1. 모바일에서 접속이 안 되는 경우
```bash
# 방화벽 확인
sudo ufw status

# 포트 확인
netstat -an | grep 3000
netstat -an | grep 5001
```

#### 2. 카카오 로그인이 안 되는 경우
- 카카오 개발자 콘솔에서 Redirect URI 설정 확인
- `http://172.16.0.91:3000/auth/kakao/callback` 추가
- JavaScript 키 확인

#### 3. API 연결 오류
- 클라이언트 환경 변수에서 API URL 확인
- 서버가 정상적으로 실행 중인지 확인
- 네트워크 연결 상태 확인

### 디버깅 도구

#### 1. 모바일 브라우저 개발자 도구
- Chrome: `chrome://inspect`
- Safari: 개발 > 웹 검사기

#### 2. 네트워크 모니터링
```bash
# 서버 로그 확인
tail -f logs/app.log

# 클라이언트 콘솔 로그 확인 (브라우저 개발자 도구)
```
