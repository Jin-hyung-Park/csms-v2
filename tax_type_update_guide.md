## 🌐 사용 방법

### 1. 회원가입
1. **http://localhost:3000** 접속
2. **회원가입** 클릭
3. **폼 작성** (세금 신고 유형 필드 없음)
4. **가입 완료** → 기본값 `미신고`로 설정

### 2. 근무 일정 승인 (점주)
1. **점주 계정으로 로그인**
2. **근무 일정 승인** → 자동으로 세금 신고 유형 업데이트
3. **주 15시간 이상**: `미신고` → `주15시간미만`
4. **주 15시간 미만**: `주15시간미만` → `미신고`

### 3. 점주가 수동 설정
1. **점주 계정으로 로그인**
2. **근로자 관리** 페이지 접속
3. **직원 선택** → 편집 버튼 클릭
4. **세금 신고 유형 수정**:
   - `미신고`: 주 15시간 미만
   - `주15시간미만`: 주 15시간 이상 (자동 설정)
   - `사업자소득(3.3%)`: 점주 수동 설정

## 📊 세금 신고 유형 설명

### 1. 미신고
- **조건**: 주 15시간 미만 근무
- **설정**: 기본값, 자동 설정
- **색상**: 회색 (default)

### 2. 주15시간미만
- **조건**: 주 15시간 이상 근무
- **설정**: 자동 설정 (근무 일정 승인 시)
- **색상**: 주황색 (warning)

### 3. 사업자소득(3.3%)
- **조건**: 점주가 수동 설정
- **설정**: 점주만 수정 가능
- **색상**: 빨간색 (error)

## 🔧 테스트 방법

### 1. 회원가입 테스트
```bash
# 새로운 사용자로 회원가입
curl -X POST http://localhost:5001/api/auth/register \
  -H "Content-Type: application/json" \
  -d '{
    "username": "testuser6",
    "email": "test6@example.com",
    "password": "password123",
    "role": "employee",
    "name": "테스트 사용자6",
    "phone": "010-1234-5681"
  }'
```

### 2. 데이터베이스 확인
```bash
# 사용자 정보 확인
mongosh convenience_store --eval "db.users.find({email: 'test6@example.com'})"
```

### 3. 근무 일정 생성 및 승인
1. **직원으로 로그인** → 근무 일정 생성
2. **점주로 로그인** → 근무 일정 승인
3. **세금 신고 유형 자동 업데이트 확인**

## 🚨 문제 해결

### 자동 업데이트가 안 되는 경우:
1. **근무 일정 상태 확인**: `approved` 상태인지 확인
2. **주간 근무 시간 계산**: 정확히 15시간 이상인지 확인
3. **서버 로그 확인**: 오류 메시지 확인

### 점주 관리 페이지 접근 불가:
1. **점주 권한 확인**: `role: 'owner'`인지 확인
2. **토큰 확인**: 유효한 JWT 토큰인지 확인
3. **API 응답 확인**: 네트워크 탭에서 오류 확인
