# 카카오 로그인 기능 테스트 가이드

## 테스트 환경 설정

### 1. 카카오 개발자 설정 확인
1. [Kakao Developers](https://developers.kakao.com/)에 로그인
2. 애플리케이션 상태가 "활성화" 상태인지 확인
3. Web 플랫폼 도메인이 `http://localhost:3000`으로 설정되어 있는지 확인
4. 카카오 로그인이 활성화되어 있는지 확인

### 2. 환경 변수 확인
```bash
# 서버 .env 파일
KAKAO_CLIENT_ID=your-kakao-client-id
KAKAO_CLIENT_SECRET=your-kakao-client-secret

# 클라이언트 .env 파일
REACT_APP_KAKAO_APP_KEY=your-kakao-javascript-key
```

### 3. 애플리케이션 실행
```bash
# 서버 실행
npm run server

# 새 터미널에서 클라이언트 실행
npm run client
```

## 테스트 시나리오

### 시나리오 1: 새로운 카카오 사용자 가입
1. 브라우저에서 `http://localhost:3000/login` 접속
2. "카카오톡으로 시작하기" 버튼 클릭
3. 카카오 로그인 팝업에서 계정 선택
4. 사용자 정보 동의 (닉네임, 이메일, 프로필 이미지)
5. 자동으로 시스템에 가입되고 로그인되는지 확인
6. 직원 대시보드로 리다이렉트되는지 확인

### 시나리오 2: 기존 카카오 사용자 로그인
1. 시나리오 1에서 가입한 계정으로 다시 카카오 로그인 시도
2. 자동으로 로그인되는지 확인
3. 사용자 정보가 올바르게 표시되는지 확인

### 시나리오 3: 기존 이메일 계정과 카카오 연동
1. 먼저 일반 회원가입으로 계정 생성 (이메일/비밀번호)
2. 로그아웃 후 카카오 로그인 시도
3. 같은 이메일로 카카오 계정 연동되는지 확인
4. 기존 계정 정보가 유지되는지 확인

### 시나리오 4: 카카오 사용자의 비밀번호 로그인 방지
1. 카카오로 가입한 계정의 이메일로 일반 로그인 시도
2. "카카오 계정으로 가입된 사용자입니다" 메시지가 표시되는지 확인
3. 카카오 로그인을 이용하라는 안내가 표시되는지 확인

### 시나리오 5: 에러 처리 테스트
1. 카카오 SDK가 로드되지 않은 상태에서 버튼 클릭
2. 적절한 에러 메시지가 표시되는지 확인
3. 네트워크 오류 시 에러 처리가 되는지 확인

## 일반적인 문제 해결

### 문제 1: "카카오 SDK가 로드되지 않았습니다"
**해결 방법:**
- JavaScript 키가 올바르게 설정되었는지 확인
- 카카오 SDK 스크립트가 HTML에 포함되었는지 확인
- 네트워크 연결 상태 확인

### 문제 2: "카카오 로그인에 실패했습니다"
**해결 방법:**
- Redirect URI 설정 확인
- 카카오 개발자 콘솔에서 앱 상태 확인
- 도메인 설정 확인

### 문제 3: "이미 등록된 이메일입니다"
**해결 방법:**
- 기존 계정과 연동 시도
- 데이터베이스에서 중복 이메일 확인

### 문제 4: 로그인 후 페이지가 로드되지 않음
**해결 방법:**
- JWT 토큰이 올바르게 발급되었는지 확인
- 리다이렉트 로직 확인
- 브라우저 콘솔에서 에러 확인
