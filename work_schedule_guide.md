# ⏰ 근로자 근무시간 입력 기능 가이드

## ✅ 구현 완료!

### 🎯 주요 기능:

#### 1. 근무시간 입력
- **날짜 선택**: 오늘 이전 날짜만 선택 가능
- **시작/종료 시간**: HH:MM 형식으로 입력
- **휴식 시간**: 0-8시간 범위에서 입력
- **메모**: 근무 중 특이사항 기록

#### 2. 근무시간 관리
- **목록 조회**: 모든 근무 일정을 테이블로 표시
- **수정**: 승인 대기 중인 일정만 수정 가능
- **삭제**: 승인 대기 중인 일정만 삭제 가능
- **상태 표시**: 승인됨, 승인 대기, 거절됨, 수정됨

#### 3. 자동 계산
- **총 근무 시간**: 시작-종료 시간에서 휴식 시간 제외
- **초과 근무**: 8시간 초과 시 자동 계산
- **실시간 미리보기**: 입력 중 예상 근무 시간 표시

## 🌐 사용 방법

### 1. 근무시간 입력
1. **http://localhost:3000** 접속
2. **직원 계정으로 로그인**
3. **근무 시간 입력** 메뉴 클릭
4. **근무 시간 추가** 버튼 클릭
5. **폼 작성**:
   - 근무 날짜: 오늘 이전 날짜 선택
   - 시작 시간: HH:MM 형식 (예: 09:00)
   - 종료 시간: HH:MM 형식 (예: 18:00)
   - 휴식 시간: 0-8시간 (예: 1.0)
   - 메모: 근무 중 특이사항 (선택사항)

### 2. 근무시간 수정
1. **목록에서 수정할 일정 선택**
2. **편집 버튼** 클릭
3. **필요한 정보 수정**
4. **저장** 버튼 클릭

### 3. 근무시간 삭제
1. **목록에서 삭제할 일정 선택**
2. **삭제 버튼** 클릭
3. **확인** 클릭

## 📊 기능 상세 설명

### 입력 필드:
- **근무 날짜**: DatePicker로 선택, 오늘 이전만 가능
- **시작 시간**: HH:MM 형식 (예: 09:00, 14:30)
- **종료 시간**: HH:MM 형식 (예: 18:00, 22:30)
- **휴식 시간**: 숫자 입력, 0-8시간 범위
- **메모**: 텍스트 입력, 최대 1000자

### 자동 계산:
- **총 근무 시간**: (종료 시간 - 시작 시간) - 휴식 시간
- **초과 근무**: 8시간 초과 시 자동 계산
- **실시간 미리보기**: 입력 중 예상 시간 표시

### 상태 관리:
- **승인 대기**: 새로 입력한 일정
- **승인됨**: 점주가 승인한 일정
- **거절됨**: 점주가 거절한 일정
- **수정됨**: 점주가 수정한 일정

## 🔧 테스트 방법

### 1. 근무시간 입력 테스트
```bash
# 직원으로 로그인 후 근무시간 입력
# 1. 근무 시간 추가 버튼 클릭
# 2. 날짜: 2025-08-04
# 3. 시작 시간: 09:00
# 4. 종료 시간: 18:00
# 5. 휴식 시간: 1.0
# 6. 메모: 정상 근무
# 7. 저장 버튼 클릭
```

### 2. API 테스트
```bash
# 근무시간 생성
curl -X POST http://localhost:5001/api/work-schedule \
  -H "Content-Type: application/json" \
  -H "Authorization: Bearer YOUR_TOKEN" \
  -d '{
    "workDate": "2025-08-04",
    "startTime": "09:00",
    "endTime": "18:00",
    "notes": "정상 근무",
    "breakTime": 1.0
  }'

# 근무시간 목록 조회
curl -X GET http://localhost:5001/api/work-schedule \
  -H "Authorization: Bearer YOUR_TOKEN"
```

### 3. 브라우저 테스트
1. **직원 계정으로 로그인**
2. **근무 시간 입력** 페이지 접속
3. **근무 시간 추가** 버튼 클릭
4. **폼 작성 및 저장**

## 🚨 문제 해결

### 입력이 안 되는 경우:
1. **로그인 상태 확인**: 직원 계정으로 로그인했는지 확인
2. **날짜 확인**: 오늘 이전 날짜인지 확인
3. **시간 형식 확인**: HH:MM 형식인지 확인
4. **브라우저 새로고침**: F5 또는 Cmd+R

### 수정/삭제가 안 되는 경우:
1. **상태 확인**: 승인 대기 상태인지 확인
2. **권한 확인**: 본인이 작성한 일정인지 확인
3. **점주 승인 확인**: 이미 승인된 일정은 수정 불가

### 계산이 잘못된 경우:
1. **시간 형식 확인**: HH:MM 형식인지 확인
2. **휴식 시간 확인**: 0-8시간 범위인지 확인
3. **날짜 확인**: 올바른 날짜인지 확인

## 💡 사용 팁

3. **메모 활용**: 특이사항은 메모에 기록
